\chapter{相关技术介绍}

\section{大语言模型技术}

LLM 是近年来自然语言处理领域的一个重要的技术方向。这类模型通常在大规模文本语料的基础上进行预训练，通过学习自然语言的统计规律和语义结构来完成语言理解和生成任务。随着模型规模愈发扩大，LLM 在文本生成、知识问答、推理分析和对话交互等任务中表现出较强的泛化能力，逐渐成为当前人工智能领域的关键基石。

LLM 的发展与深度学习技术的进步密切相关。早期的语言模型主要依赖循环于神经网络和长短期记忆网络，但是，它在处理长距离依赖和复杂语义时存在着明显的不足。Transformer 架构的提出改变了这一局面。该架构通过自注意力机制捕捉文本中的长距离依赖关系，使得模型能够在更大规模的数据和参数条件下处理复杂的语义，表现出强大的表达能力。基于 Transformer 的预训练模型，如 GPT 系列、BERT 系列和 LLaMA 系列等，推动了 LLM 技术的快速发展。

在预训练阶段， LLM 通常采用自回归或者自编码的方式学习语言分布。自回归模型通过预测下一个词来学习语言生成能力，而自编码模型通过填补缺失词来学习语义理解能力。在此基础上，模型可以通过指令微调、对齐训练等方法进一步提升其在特定任务中的表现，使模型能够更好地理解用户意图，并生成更加符合任务需求的回答。

得益于大规模参数和丰富的语料库， LLM 具备较强的语言生成能力。模型能够根据用户的输入生成连贯、自然的文本，并在一定程度上完成推理、总结、解释和知识检索等任务。这些特点使其在教育、科研、内容创作和智能服务等场景中具有较高的应用潜力。

但是， LLM 也存在一些局限。首先，由于模型的知识来源于预训练语料，它的回答可能与真实教材、课程内容或者最新知识不一致。其次，模型可能生成“幻觉”。此外，模型缺乏对外部知识的实时访问能力，难以保证回答的可追溯性和一致性。这些问题会影响 LLM 在需要高准确性的场景中的可靠性。

在教育场景中， LLM 能够以自然语言方式理解学生提出的问题，并生成解释性的回答，从而在一定程度上缓解教师的答疑压力。但是，课程内容具有明确范围和严格要求，仅依赖通用大语言模型难以保证回答的准确性、教学一致性和内容可验证性。为了提升可靠性，通常需要结合检索增强生成技术，将模型与课程知识库结合，使其能够在回答问题时引用真实资料并减少幻觉。

\section{检索增强生成技术}
RAG 技术是一种将大语言模型与外部知识库结合的技术框架，通过在生成最终回答之前引入一个检索步骤，使模型能够访问与输入问题相关的真实语料库，从而提升回答的准确性、可追溯性和一致性，适用于教育、医疗和法律等对内容准确性要求较高的任务。

在技术流程上，RAG 通常包括三个主要步骤。第一步是查询构建：系统根据用户输入生成检索查询，并将其转换为向量表示。第二步是文档检索：系统使用向量检索或关键词检索方法，从知识库中找到与查询最相关的文档片段。第三步是答案生成：大语言模型在生成回答时会结合检索到的内容，使回答更准确、更可解释。通过这种方式，模型能够减少幻觉，并保持与领域知识的一致性。

RAG 技术的优势主要体现在三个方面。第一，系统能够引用真实资料，从而提升回答的可靠性。第二，系统能够保持与课程内容或领域知识的一致性，减少模型因为预训练语料的差异所带来的偏差。第三，系统能够支持内容更新，只需更新知识库即可使模型获得最新信息，而无需重新训练模型。

但是，RAG 也存在一些挑战。检索结果的质量会直接影响生成内容的质量。如果检索不到相关资料，模型仍可能出现幻觉。此外，知识库构建和维护需要额外成本，课程资料的格式、结构和更新频率也会影响系统的整体效果。因此，在实际应用中，需要对知识库进行清洗、分段和索引优化，以保证检索质量。

在教育场景中，RAG 技术能够有效弥补通用大语言模型的不足。学生问题通常依赖具体教材、实验步骤和课堂内容，而通用模型难以保证与这些内容保持一致。通过引入课程知识库，RAG 能够使系统在回答问题时引用真实资料，从而提升准确性和教学一致性。这种方式不仅减少教师答疑压力，也为构建可控、可靠的智能助教系统提供了可行路径。
